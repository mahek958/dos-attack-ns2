# Web Application Penetration Testing – OWASP Juice Shop

## 📌 Project Overview
This project demonstrates a complete **web application penetration testing process** performed on the intentionally vulnerable application **OWASP Juice Shop**.  

The objective of this project is to simulate a real-world penetration testing engagement by identifying, analyzing, and exploiting vulnerabilities in a controlled lab environment.

The project follows an industry-standard penetration testing methodology including reconnaissance, vulnerability discovery, exploitation, and reporting.

---

## 🎯 Objectives
- Understand the fundamentals of web application security
- Identify common vulnerabilities based on **OWASP Top 10**
- Perform manual and automated security testing
- Analyze the security impact of discovered vulnerabilities
- Document findings and recommend mitigation strategies

---

## 🛠 Tools & Technologies Used
- **Kali Linux**
- **Burp Suite**
- **Nmap**
- **OWASP Juice Shop**
- **Browser Developer Tools**

---

## 🔍 Testing Methodology

The penetration testing process followed these stages:

### 1️⃣ Reconnaissance
Information gathering about the target application including:
- Domain and server identification
- Open ports and services
- Application structure analysis

### 2️⃣ Vulnerability Scanning
Using security tools and manual testing to detect vulnerabilities such as:
- SQL Injection
- Cross-Site Scripting (XSS)
- Broken Authentication
- Security Misconfiguration

### 3️⃣ Exploitation
Attempting to exploit identified vulnerabilities to demonstrate their impact on the system.

### 4️⃣ Post Exploitation & Analysis
Understanding the consequences of the exploited vulnerabilities and analyzing potential risks.

### 5️⃣ Reporting
Documenting:
- Vulnerability details
- Impact analysis
- Recommended mitigation techniques
